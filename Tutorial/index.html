DR Sites

1) Add DR Site

To add a new DR Site, open the PVEASS dashboard and navigate to the "DR Sites" section.

Screenshot 1: Adding a new DR Site

After filling in the details, click Add DR Site. The new DR Site will now appear in the list.

2) Associate Proxmox Backup Server

Associate your DR Site with a Proxmox Backup Server (PBS) to enable restore from backup.

Screenshot 2: Associating PBS with DR Site

3) Create ZFS in DR Site

In the DR Site, create a ZFS storage pool to store VM and container data.

Screenshot 3: Creating ZFS in DR Site

4) Add API Tokens

Generate API Token ID and Secret in Proxmox VE and add API tokens for secure communication between DR Site and PBS.

Screenshot 4: Adding API Tokens

5) Restore VM/Container in DR Site

Use the Restoration option to bring back a VM or container from backup.

Screenshot 5: Restoring a VM in DR Site

How to in Proxmox VE

1) Creating API Token and assigning Permission

In Proxmox VE, create an API token under Datacenter → Permissions → API Tokens.

Screenshot 6: Creating API Token in Proxmox VE

2) Creating Datastore in Proxmox Backup Server

In PBS, create a new datastore to hold VM/container backups.

Screenshot 7: Creating datastore in PBS

3) View mounted Storage (PBS for Backup) and ZFS (for VM/Container) in DR Site

In the DR Site host, verify that PBS is mounted for backups and ZFS and PBS Storage are available for VM/Container storage and Restoration.

Screenshot 8: Viewing mounted storage in DR Site
